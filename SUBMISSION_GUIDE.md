# 提交整理说明

本文档用于整理课程项目的最终提交内容、演示顺序和截图清单。

## 1. 建议提交结构

建议分成两个压缩包提交：

### `source.zip`

包含源码、说明文档和必要资源，不包含 `build/` 中的大量中间文件。

建议目录：

```text
Image-Processing-Platform/
├─ src/
├─ installer/
├─ 图片/
├─ CMakeLists.txt
├─ README.md
├─ RUN_RELEASE.md
└─ SUBMISSION_GUIDE.md
```

### `release.zip`

包含可以直接运行的程序目录。

建议目录：

```text
release/
└─ MedicalImageAnalysisPlatform/
   ├─ MedicalImageAnalysisPlatform.exe
   ├─ Qt6Core.dll
   ├─ Qt6Gui.dll
   ├─ Qt6Widgets.dll
   ├─ ...
   └─ 其他 Qt 运行库
```

说明：

- 不建议只提交单个 `.exe`
- 运行包直接取自 `build/Release/`
- 源码包和运行包分开，老师更容易检查

## 2. 提交前最终检查

提交前建议逐项确认：

1. `cmake --build build --config Release` 可以成功通过
2. `build/Release/MedicalImageAnalysisPlatform.exe` 可以正常启动
3. 可以打开一套 DICOM 图像
4. 可以显示轴状位、冠状位、矢状位
5. 可以调节窗宽窗位
6. 可以运行算法分割
7. 可以导入 DICOM SEG
8. 可以显示分割叠加结果
9. 可以执行图像增强功能
10. 可以显示三维重建结果
11. 三维视图中可以看到二维切面位置
12. 旋转三维模型时无明显功能性错误

## 3. 建议答辩演示顺序

建议严格按下面顺序演示，最稳。

### 第一步：介绍界面结构

重点说明：

- 菜单栏
- 工具栏
- 左侧文件列表区
- 中间图像显示区
- 右侧信息与参数区
- 下方实验工作台

### 第二步：演示 DICOM 打开与显示

操作顺序：

1. 打开 DICOM 文件夹或单个 DICOM 文件
2. 展示轴状位、冠状位、矢状位
3. 调整窗宽窗位
4. 展示状态栏和信息区变化

讲解重点：

- 软件能够完成 DICOM 图像读取、解读、显示
- 支持医学图像多视图浏览

### 第三步：演示分割功能

建议演示两条路径：

#### 路径 A：算法分割

1. 选择阈值分割
2. 选择 Otsu 自动阈值分割
3. 如有需要再演示区域生长
4. 点击运行分割
5. 展示二维叠加结果

#### 路径 B：导入 DICOM SEG

1. 导入 SEG 文件
2. 展示分割结果自动对齐到 CT 序列
3. 展示三视图中的叠加显示

讲解重点：

- 软件支持分割算法
- 软件支持外部分割结果导入
- 分割结果可视化完整

### 第四步：演示图像增强

建议演示以下任意 2 到 3 个功能：

- 直方图均衡化
- 线性拉伸
- 均值/中值滤波
- Sobel / Laplacian
- Gamma 变换

讲解重点：

- 软件具有医学图像增强处理能力
- 功能来源于过往实验，并已整合到平台中

### 第五步：演示三维重建

操作顺序：

1. 在存在分割结果的前提下打开三维视图
2. 等待预览重建结果出现
3. 稍后展示高质量三维模型替换
4. 旋转、缩放、平移模型

讲解重点：

- 软件支持三维重建
- 重建结果可交互显示
- 已做异步预览和完整重建优化

### 第六步：演示三维视图中二维切面位置

操作顺序：

1. 调整 axial / coronal / sagittal 切片
2. 观察三维视图中的三色切面位置变化
3. 指出图例对应关系

讲解重点：

- 三维视图中能显示二维图像所在空间位置
- 可体现二维与三维的联动关系
